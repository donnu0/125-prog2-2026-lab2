# Команда pwd (Варіант 7)

### 1. Що робить ваша команда?
Команда `pwd` (Print Working Directory) призначена для виведення повної абсолютної траєкторії поточної робочої директорії, у якій перебуває користувач в командній оболонці `minish`.

### 2. Які некоректні аргументи ви обробляєте?
Оскільки базова команда `pwd` не потребує параметрів, реалізовано принцип "Zero Trust": якщо довжина списку `argv` більша за 1 (користувач ввів щось на кшталт `pwd -xyz` чи `pwd /etc`), програма виводить інструкцію `Usage: pwd` у потік помилок `stderr` і повертає код 2.

### 3. Що може піти не так під час виконання?
Можливий збій під час звернення до полів об'єкта контексту `ctx` або відсутність прав доступу до поточної папки в операційній системі. Будь-які системні винятки перехоплюються Couldn't-block `try-except` для запобігання аварійного завершення (краху) всього shell.

### 4. Які коди завершення ви повертаєте і чому?
* **0** — Успіх: шлях успішно зчитано та виведено в `stdout`.
* **1** — Помилка виконання: внутрішній системний збіг.
* **2** — Помилка використання: передано некоректні/зайві аргументи.
