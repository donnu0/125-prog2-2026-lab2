# Команда id

## Що робить команда?

`id` без аргументів друкує інформацію про поточного користувача shell:

```text
uid=1000 gid=1000 user=anna
```

На Windows, де `os.getuid()` і `os.getgid()` зазвичай недоступні, команда не падає і виводить:

```text
uid=N/A gid=N/A user=Anna
```

Ім'я користувача визначається через `getpass.getuser()`.

## Які некоректні аргументи обробляються?

Команда не підтримує аргументи або параметри. Будь-який виклик на кшталт `id test`, `id --bad` або некоректний `argv` друкує usage у stderr:

```text
usage: id
```

і повертає код `2`.

## Що може піти не так?

Під час виконання може статися внутрішня помилка отримання імені користувача або системних ідентифікаторів. Такі винятки перехоплюються, stack trace не друкується, а керування повертається shell. Якщо `uid` або `gid` неможливо отримати, для них використовується `N/A`.

## Коди завершення

* `0` - успішне виконання.
* `1` - внутрішня помилка виконання.
* `2` - помилка використання або некоректні аргументи.
