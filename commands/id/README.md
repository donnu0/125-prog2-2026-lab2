# Команда id

## Що робить команда?

Команда `id` показує інформацію про поточного користувача. Її треба запускати без аргументів.

```text
uid=1000 gid=1000 user=anna
```

Якщо система не підтримує `uid` або `gid`, наприклад на Windows, команда не завершується з помилкою. У такому випадку вона виводить `N/A`:

```text
uid=N/A gid=N/A user=Anna
```

## Які некоректні аргументи обробляються?

Команда не приймає додаткові аргументи. Якщо написати `id test`, `id --bad` або передати інші зайві дані, команда виведе підказку в потік помилок:

```text
usage: id
```

Після цього вона поверне код `2`.

## Що може піти не так під час виконання?

Може не вийти отримати ім'я користувача або системні номери `uid` і `gid`. Для `uid` та `gid` у такому випадку використовується `N/A`. Якщо станеться інша внутрішня помилка, команда не друкує службовий traceback, а повертає керування shell.

## Які коди завершення повертаються і чому?

* `0` - команда виконалась правильно.
* `1` - сталася внутрішня помилка під час виконання.
* `2` - команду викликали неправильно, наприклад із зайвими аргументами.
