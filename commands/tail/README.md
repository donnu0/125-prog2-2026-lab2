# Команда tail

**Що робить ваша команда?**
Команда `tail` виводить останні рядки вказаного текстового файлу у стандартний потік виводу (`stdout`). За замовчуванням виводиться 10 рядків. Підтримується прапорець `-n` для вказівки точної кількості рядків (наприклад, `-n 5`). Для оптимізації використання пам'яті та уникнення завантаження всього файлу застосовано кільцевий буфер (`collections.deque`).

**Які некоректні аргументи ви обробляєте?**
Обробляється відсутність будь-яких аргументів, відсутність імені файлу або відсутність значення після прапорця `-n`. Також передбачено обробку нечислових або від'ємних значень для аргументу `-n`. При некоректних аргументах у потік `stderr` виводиться usage-повідомлення.

**Що може піти не так під час виконання?**
Файл може не існувати (`FileNotFoundError`), користувач може не мати прав на його читання (`PermissionError`), або файл може бути недоступний через інші системні причини (`OSError`). Крім того, передбачено глобальний блок `try-except` для перехоплення непередбачуваних внутрішніх винятків Python, щоб гарантувати стабільність оболонки (Zero Trust).

**Які коди завершення ви повертаєте і чому?**
* `0`: Успішне читання та вивід файлу.
* `1`: Помилки виконання (файл не знайдено, відмовлено в доступі, внутрішня програмна помилка).
* `2`: Некоректне використання команди (неправильні аргументи, невалідне число для `-n`).
